# 👁️ VLA 发帖规范 (Vision-Language-Action Rules)

**存放目录**：`posts/vla/` 或其相关的多级嵌套下级子目录（如 `posts/vla/vla-2025/`）  
**首页挂载 `index.html` 的 `data-category` 属性值**：`vla`

## 内容范畴要求
专属前沿跟踪领域：一切关于 Vision-Language-Action (视觉-语言-动作) 端到端模型的最新论文解读（如 OpenVLA, π0, GROOT, Octo 等）。关注具身大模型的多模态推理和直接输出执行能力的演进路线与模型特点。

## 版块参考标签 (Tag)
生成本类文章和对应首页卡片时，以下是高频标签参考，你**完全可以不被其限制**，根据模型特点自由拓展高度定制化的词汇：
- `论文解读`
- `前沿动态`
- `模型评测`
- `多模态`
- `VLA架构`

## 专项排版与内容质量期望
关于人工智能前沿动态的文章极易因“幻觉”而丧失学术价值。因此撰写 VLA 综述与论文解析时要求极其严厉的约束：
1. **不可撼动的客观溯源原则，拒绝“编造实验”**：提及模型的评估数值（如“在 CALVIN 基准测试上提升了 13%”），一律不可擅自捏造，必须确有其事！此外页眉或结尾**必须**附上引用的论文真实链接（如 `arxiv`），或者提供开源模型的 `Hugging Face / GitHub` 准确连接以供验证。
2. **拒绝蜻蜓点水，推崇细节拉满**：AI 撰写 VLA 模型分析时，内容应当详尽且有穿透力。分析时必须带出具体使用了哪种 Backbone（视觉端如 SigLIP，语言端如 Llama），它的 Action 头是怎么接出来的？使用了扩散机制还是自回归机制？
3. **拆解阅读模式（八股文标准）**：为提升读者获取信息的效率，一律采用以下逻辑严密的章节进行分块解析：
   - **痛点与背景**：明确前人研究的死穴，以及本研究解决什么核心阻碍。
   - **核心方法 (Methodology)**：他们用了什么突破或整合型的创新架构。
   - **实验验证与指标**：列出实测数据（需真实），超越了什么基线模型（Baselines）。
   - **局限性分析**：不可刻意美化、隐瞒模型尚未解决的缺陷。
4. **系列归档逻辑**：如果是属于统揽大类目的巨型综述下拆分的单篇子论文（如作为 2025 综述的一部分），生成的单篇文章应当放置于 `posts/vla/vla-2025/` 子文件夹中，同时页面左上方的导航按钮需确保指向上层综述（`../vla-2025.html`）而非强行退回最外层主页。
