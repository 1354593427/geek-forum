🪟 Windows (WSL2) - 官方推荐
⚠️ 强烈推荐使用 WSL2

OpenClaw 官方强烈推荐在 Windows 上使用 WSL2 (Windows Subsystem for Linux 2)。原生命令行支持不完善，WSL2 可提供与 Linux 一致的体验，避免路径、权限、工具兼容性问题。

03 关键配置详解

🔍 为什么要求这些配置？

1. Windows 版本要求 (WSL2)

  • 原因：WSL2 仅在 Windows 10 版本 2004 (Build 19041) 及以上版本可用
  • 查询方法：设置 → 系统 → 关于，查看 "Windows 规格"
  • 不符合条件：Win10 1903/1909 用户需要手动升级或使用 WSL1（不推荐）

2. 内存 (RAM) 4GB 起步

  • 原因：Node.js 22 运行时需要约 100-200MB 基础内存
  • Gateway 消耗：OpenClaw Gateway 后台服务需要 200-500MB 内存
  • 操作系统开销：Windows/macOS/Linux 本身需要约 2-3GB 内存
  • 建议：如果同时运行其他应用，8GB 更舒适

3. CPU 虚拟化支持

  • 原因：WSL2 需要 CPU 虚拟化技术 (Intel VT-x 或 AMD-V)
  • 检查方法：
    • Intel: 控制面板 → 程序 → 启用/关闭 Windows 功能 → 勾选 "Hyper-V"
    • AMD: 已在 AMD-V 中默认启用
  • BIOS 设置：如遇问题，需在 BIOS 中开启 Virtualization Technology

4. 磁盘空间

  • WSL2 Ubuntu 基础镜像：约 5-8GB
  • Node.js + OpenClaw 全局安装：约 500MB - 1GB
  • 日志和缓存：随着使用会增加，建议预留空间
  • SSD vs HDD：强烈推荐 SSD，IO 性能对 Node.js 开发体验影响很大

04 快速自查清单

安装前检查以下项目：

Windows 用户：

  • ☑️ Windows 10 版本 ≥ 2004 或 Windows 11
  • ☑️ CPU 支持虚拟化技术 (VT-x/AMD-V)
  • ☑️ 已开启 Hyper-V 或 WSL 功能
  • ☑️ 有 20GB+ 可用磁盘空间

macOS 用户：

  • ☑️ macOS 版本 ≥ 10.15
  • ☑️ 有 10GB+ 可用磁盘空间

Linux 用户：

  • ☑️ Ubuntu 18.04+ 或 Debian 10+
  • ☑️ systemd 已启用
  • ☑️ 有 10GB+ 可用磁盘空间

05 安装方式

方式一：Installer 脚本（推荐）

macOS / Linux / WSL2:

curl -fsSL https://openclaw.ai/install.sh | bash

Windows (PowerShell):

iwr -useb https://openclaw.ai/install.ps1 | iex
方式二：npm / pnpm 手动安装
npm install -g openclaw@latest
openclaw onboard --install-daemon

06 验证安装与环境自检

安装完成后，为了确保 OpenClaw 后台守护进程（Daemon）正常运转并能与您的宿主机硬件正确通信，必须进行严格的生命周期验证。请在您的终端中输入并执行以下基础自检命令：

# 1. 检查 OpenClaw 核心服务的运行状态与健康度
openclaw status

# 2. 检查当前安装的 CLI 客户端与 Daemon 服务端版本是否匹配
openclaw --version

✓ 预期的正常输出示例 (Expected Output)

● OpenClaw Daemon is active and running
└─ CLI Version: 2.1.0-stable
└─ Daemon Platform: linux-x64 (WSL)
└─ Underlying Node.js: v22.0.0
└─ Port Listening: :8080
💣 常见自检错误排查 (Troubleshooting)
  • 命令找不到 (command not found)：意味着 npm 的全局二进制路径尚未注册。请检查 Node.js 全局安装路径（如 ~/.npm-global/bin）是否已正确追加至您的 ~/.bashrc 或 ~/.zshrc 的 $PATH 变量中。
  • Daemon 启动崩溃 (EACCES: permission denied)：通常发生于 Linux/macOS 环境下无特权用户的权限不足。请尝试执行 sudo openclaw onboard --repair 修复运行时权限。
  • 端口地址冲突 (EADDRINUSE: Port 8080)：OpenClaw Gateway 默认监听宿主机的 8080 端口。若已被其他服务（如 Nginx, Tomcat）占用，必须手动映射：可通过注入环境变量 export OPENCLAW_PORT=9090 来强制切换服务的监听端口。
