OpenClaw 安装配置最低要求

macOS / Linux / Windows (WSL2)

01 系统要求

配置项 最低要求 备注
Node.js 22.0.0+ installer 脚本会自动安装
操作系统 macOS / Linux / Windows Windows 推荐 WSL2
pnpm 仅源码构建时需要 npm 安装方式不需要
网络 需要访问 GitHub / npm 国内需配置代理
⚠️ Windows 用户特别说明

官方强烈推荐在 Windows 上使用 WSL2 (Windows Subsystem for Linux) 运行 OpenClaw。虽然也有原生 Windows 支持，但 WSL2 可以避免很多路径和权限相关的问题。

02 安装方式

方式一：Installer 脚本（推荐）

一行命令自动完成 Node 检测、安装和配置向导。

macOS / Linux / WSL2:

curl -fsSL https://openclaw.ai/install.sh | bash

Windows (PowerShell):

iwr -useb https://openclaw.ai/install.ps1 | iex

跳过配置向导（静默安装）：

curl -fsSL https://openclaw.ai/install.sh | bash -s -- --no-onboard
方式二：npm / pnpm 手动安装

如果你已经安装了 Node 22+，可以使用 npm 或 pnpm：

npm:

npm install -g openclaw@latest
openclaw onboard --install-daemon

pnpm:

pnpm add -g openclaw@latest
pnpm approve-builds -g
openclaw onboard --install-daemon

pnpm 需要手动批准包含构建脚本的包，第一次安装后运行 pnpm approve-builds -g 并选择列出的包。

方式三：从源码构建

适合贡献者或需要自定义的场景：

git clone https://github.com/openclaw/openclaw.git
cd openclaw
npm install
npm run build
npm link

03 三大平台安装对比

✅ macOS

  • 安装最简单
  • 原生支持完善
  • 推荐 Homebrew 安装 Node

⚠️ 注意事项

  • 如遇 sharp 构建错误，设置 SHARP_IGNORE_GLOBAL_LIBVIPS=1

✅ Linux (Ubuntu/Debian)

  • 与 macOS 类似
  • 安装脚本自动处理
  • 推荐使用 installer

⚠️ 注意事项

  • 确保 npm 全局 bin 目录在 PATH 中

✅ Windows (WSL2)

  • 官方推荐方案
  • 避免路径/权限问题
  • 与 Linux 体验一致

⚠️ 注意事项

  • 需要 WSL2 安装和配置
  • 不推荐原生 Windows

04 验证安装

安装完成后，验证 OpenClaw 是否正常运行：

openclaw status

查看版本：

openclaw --version
